LLM-Driven Cold Start Recommendation System

Python
Pandas
NumPy
PyTorch
NetworkX
MPNet V2
Collaborative Filtering
Knowledge Graph
Graph Retrieval


This project introduces a hybrid movie recommendation system that combines knowledge graphs with Large Language Models (LLMs) to effectively solve the cold start problem-making relevant suggestions for users with little or no history. Using the MovieLens 100K dataset, the system builds user profiles by blending statistical genre preferences with LLM-driven semantic insights, then retrieves and ranks candidate movies through graph traversal and language model reasoning. Compared to traditional collaborative filtering and graph-only approaches, the LLM+Graph method delivers up to 20% higher precision, 37% better recall, and more diverse, explainable recommendations, all while remaining computationally efficient-even with lightweight models like DistilGPT2. The result is a transparent, accurate, and scalable recommender that excels for both new and established users, providing natural language explanations for each suggestion and demonstrating state-of-the-art performance in cold start scenarios.

Zoomception: CLIP Model Visualization project, is an interactive web-based platform designed to make the inner workings of OpenAI’s CLIP vision-language model transparent and accessible. By integrating advanced data visualization techniques-including 3D embedding plots, similarity heatmaps, and hierarchical model architecture trees-the tool enables users to intuitively explore how CLIP aligns /images and text in a shared embedding space and how its attention mechanisms operate across modalities. The project not only bridges the gap between AI model complexity and human interpretability but also lays the groundwork for future enhancements, such as semantic clustering and support for additional vision-language models.

COVID-19 Vaccination Portal

Developed and deployed a full-stack web portal to efficiently manage the vaccination statuses of 2,500+ engineering students, reducing lookup time by 80% and supporting streamlined campus health compliance. Initially built and tested using XAMPP, the system was later migrated to AWS, leveraging EC2 for compute, RDS for MySQL database, and S3 for static content storage, resulting in a 50% improvement in load times and enhanced accessibility.
